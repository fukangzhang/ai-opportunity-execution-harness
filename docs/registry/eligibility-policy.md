# 参赛资格规则

## 核心规则

我们按从业人员身份参赛。凡是公开规则明确限制在读学生参赛的比赛，直接排除。

学生限定比赛不进入：

- `competition_matrix.csv`
- `docs/registry/competition-index.md`
- `competitions/10-candidates/`
- `competitions/20-active/`
- `competitions/30-watchlist/`

## 资格标签

| 标签 | 含义 | 处理方式 |
|---|---|---|
| `开放参赛` | 个人或团队可报名，不限制学生身份 | 可作为主线 |
| `企业/组织限定` | 需要公司、机构或组织身份 | 立项前核验 |
| `邀请制` | 需要筛选、邀请或入营 | 未入选前放入观察 |
| `未知` | 规则未核验 | 进入 active 前必须核验 |

## 排除流程

发现比赛后先查公开规则：

1. 如果确认限制在读学生参赛，停止录入。
2. 如果已经进入矩阵或索引，删除对应条目。
3. 如果只是比赛名称出现在来源索引、短名单、行动清单中，也一并删除。
4. 只有资格不限制从业人员时，才继续核验截止时间、交付物和 8GB 可行性。

## 记录原则

不为学生限定比赛建立任何比赛条目。这样做是为了保持仓库干净，让比赛库始终服务于“我们能正式参赛”的目标。
