# 方向雷达：应用、时序、气象、多模态

调研日期：2026-05-14。  
环境约束：Windows + 8GB 显存。  
用途：后续选赛时快速判断“该不该主攻、怎么切入、需要避开什么坑”。

## 1. AI 应用 / Agent / RAG

优先级：最高。

适合理由：

- 多数应用赛评分不只看 leaderboard，更看真实问题、可用产品、演示、创新性和落地价值。
- 8GB 本地显存可以承担前后端、RAG、检索、数据处理、评测、演示；大模型能力可用 API 或平台能力补足。
- 作品模板可复用：法律、政务、教育、企业数据分析、交通、制造、气象服务都可以套同一套 Agent/RAG 工程底座。

推荐赛源：

- Proof of Usefulness: Tech & AI Hackathon。
- Google Cloud Rapid Agent Hackathon。
- 科大讯飞 AI 开发者大赛。
- 华为云杯人工智能 OPC 应用创新大赛。
- 第二届综合交通运输大模型智能体创新应用大赛。

参考但不主攻：

- Microsoft Agent Academy Hackathon：Agent 材料和评分项可参考，但至少要求使用一个 Microsoft 产品，偏低代码/企业生态展示，不作为第一主线。

主攻策略：

- 先做“可试用产品”，再补算法亮点。
- 每个作品固定包含：问题定义、目标用户、数据/知识库、核心流程、Agent 工具、评测指标、失败兜底、演示视频。
- 技术上优先使用 API 模型、embedding、小型 reranker、轻量数据库和可部署 Web UI。

## 2. 时序 / 表格 / 传统机器学习

优先级：高，适合作为算法练手和稳定拿分方向。

适合理由：

- 8GB 显存完全够用，很多赛题 CPU + LightGBM/XGBoost/CatBoost 就能做出强 baseline。
- 能训练比赛基本功：EDA、交叉验证、防泄露、特征工程、提交规范、实验记录。
- 对应用赛也有迁移价值，例如销量预测、风险预警、设备故障、交通流量、能源需求。

推荐赛源：

- Kaggle Playground / Getting Started。
- Kaggle Store Sales - Time Series Forecasting。
- Kaggle Predicting F1 Pit Stops。
- DrivenData DengAI、Flu Shot、Pump it Up。
- DataFountain、天池、CCF BDCI 的表格/图算法/时序类赛题。

主攻策略：

- 第一阶段只做传统 ML：LightGBM、XGBoost、CatBoost、sklearn。
- 时序题优先做稳健验证：时间切分、滚动验证、节假日/天气/滞后特征。
- 深度时序模型只在传统方法到瓶颈后尝试，避免陷入调参和显存消耗。

## 3. 气象 / 气候 / 地理空间 AI

优先级：中高，作为“应用 + 时序 + 多源数据”的复合方向跟踪。

适合理由：

- 气象方向有真实社会价值，适合做 AI 应用：预警解释、天气风险助手、可视化平台、行业决策工具。
- 算法上可从轻量时序、后处理、可视化、检索问答切入，不一定要训练大型天气模型。

风险：

- 端到端 AI 天气预报、全球网格预测、雷达临近预报通常需要专业数据、专业评估和较强算力。
- 中国气象局示范计划、ECMWF AI Weather Quest 更偏专业团队和研究/部署系统，个人 8GB 环境不适合直接主攻。

推荐赛源和参考：

- ECMWF Code for Earth 2026：应用/开源项目型，适合复盘下一届。
- ECMWF AI Weather Quest：专业气象 ML 竞赛，适合学习评估框架和问题定义。
- GeoAI Challenge 2026：地理空间 AI 应用方向。
- ITU AI and Space Computing Challenge：遥感、作物、水质、城市热岛。
- 中国气象局人工智能气象预报模型示范计划：专业示范方向，跟踪行业趋势。

可做切入：

- 气象服务应用：面向用户/行业的风险解释、预警摘要、辅助决策。
- 轻量预测：站点级降雨、温度、风速、能耗/产量与天气关联预测。
- 多源可视化：天气、遥感、交通、农业、水质等数据融合展示。

## 4. 多模态 / 视觉 / 音频 / VLM

优先级：中，偏学习和备选，不作为第一主线。

适合理由：

- 多模态是热点，很多赛题有应用价值：文档理解、图文审核、音频识别、遥感、工业视觉、智能交互。
- 可以采用 API/VLM 推理、现成 embedding、小模型分类、特征提取 + 传统模型，做出可交付原型。

风险：

- 需要大规模图文模型训练、视频模型、VLM 高效推理优化、AI 芯片部署、具身机器人抓取的赛题，不适合 8GB Windows 主攻。
- 多模态赛题的数据清洗和标注理解成本高，容易比想象中更耗时。

推荐赛源和参考：

- Kaggle BirdCLEF+：音频识别，可用平台 Notebook 做 baseline。
- 数字中国数据应用赛道：包含多模态伪造分析等方向。
- DataFountain 多模态感知融合增强的具身自主抓取：学习参考，不主攻。
- IEEE AICAS 2026 VLM 高效推理与优化：学习参考，不主攻。
- 科大讯飞 AI 开发者大赛：语音、多模交互和办公娱乐应用更适合切入。

可做切入：

- 文档多模态：PDF/图片/表格理解 + RAG。
- 图文审核：小模型/API + 规则 + 人机协同。
- 音频应用：音频转写、检索、分类、摘要。
- 遥感/地理空间：用平台算力做特征提取，本地做应用和后处理。

## 综合优先顺序

| 顺位 | 方向 | 是否主攻 | 原因 |
|---|---|---|---|
| 1 | AI 应用 / Agent / RAG | 是 | 与个人方向最匹配，8GB 友好，可复用模板 |
| 2 | 表格 / 时序 / 传统 ML | 是 | 可本地完成，适合积累算法比赛能力 |
| 3 | 气象应用 / 地理空间 AI | 备选主攻 | 有价值，但要避开重模型天气预报 |
| 4 | 多模态应用 | 备选 | 用 API/小模型可做，端到端训练不适合 |
| 5 | VLM 优化 / AI 芯片 / 具身机器人 | 否 | 算力、硬件和工程门槛高 |
