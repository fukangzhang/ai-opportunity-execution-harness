# Storyboard

## 0.0-8.8 Hook

Concept: The viewer starts with the core pain: choosing the wrong AI opportunity costs more than implementation. The frame feels like a decision desk, with proof-oriented metrics visible before any UI details.

Visuals: Large editorial headline, utility stats, evidence language, warm grid texture.

## 8.0-20.8 Hidden Constraints

Concept: The opportunity page is not a simple brief. The useful parts are scattered across rules, deadlines, platform expectations, and evidence demands.

Visuals: Four constraint tiles: eligibility, deadline, platform, evidence.

## 20.0-34.0 Structured Card

Concept: We show the real app screenshot and make the structured recommendation concrete.

Visuals: Proof of Usefulness card on the left, UI screenshot on the right, slow image motion.

## 33.5-45.3 Counterexample

Concept: The product is most useful when it says no or downgrade. Microsoft Agent Academy appears open, but the architecture would not become reusable code for this project.

Visuals: Looks open vs not reusable split frame.

## 44.8-55.4 Beyond Competitions

Concept: The harness is a general opportunity filter, not just a competition tracker.

Visuals: Three-step internal RAG support bot flow.

## 54.8-66.5 Audit Trail

Concept: Proof of usefulness comes from repeatable artifacts: input, schema, JSON, run logs, evidence.

Visuals: Code-like audit trail and saved run log screenshot.

## 65.8-75.0 CTA

Concept: Close on the durable principle: define evidence before building.

Visuals: Product name, deployment placeholders, GitHub placeholders.
